## v1.6.1 — 修復 v1.6 升版時遺失的希望配達日期功能（2026/06/19）

### 性質說明
本次為**修復**，非新功能開發。v1.6 升版過程中，因誤用舊版 index.html 為基底，
導致 v1.5.5 已完成的「希望配達日期」前台功能遺失，本版修復此問題。

### 問題
v1.6 開發「不可取日控制」時，誤使用缺少「希望配達日期」功能的舊版 `index.html` 作為基底，導致：
- ❌ 希望配達日期輸入欄位消失
- ❌ 配達日期寫入 payload 消失
- ❌ 前台確認摘要顯示消失

### 確認未受影響
- ✅ `admin.html` 保留配達日期顯示程式
- ✅ `Code.gs` 保留 V欄寫入邏輯
- ✅ Google Sheet 欄位結構未變更

### 開發原則
以 v1.6 為主版本，保留：
- 前台：不可取日控制
- 後台：今日營收統計

補回 v1.5.5 功能（index.html）：
- ✅ 希望配達日期輸入欄位
- ✅ 顯示／隱藏邏輯（宅配顯示／7-11店到店顯示／自取隱藏）
- ✅ 訂單確認摘要顯示
- ✅ payload 寫入 Google Sheet V欄

### 修改檔案
- `index.html`

### 測試項目

**前台**
- [x] 自取時不顯示配達日期
- [x] 宅配時顯示配達日期
- [x] 7-11店到店顯示配達日期
- [x] 不可取日無法選取

**Google Sheet**
- [x] V欄成功寫入配達日期

**後台**
- [x] 訂單卡片顯示配達日期
- [x] 今日營收統計正常

**整體驗證**
- [x] 配達日期功能正常（已於正式網址截圖確認）
- [x] 舊功能無回歸問題（diff 比對僅新增10行，無刪除）
- [x] 不可取日控制正常

### 經驗記錄
這是「develop/main 分支不同步」問題的變形——本次根因不是分支沒合併，而是**新版本開發時引用了舊版檔案做基礎，導致已完成功能被覆蓋遺失**。建議之後每次升版前，用關鍵字比對新舊版本，確認沒有不小心漏掉既有功能。

## 開發過程重要決策記錄

### 技術選型
- **為何選 Google Apps Script**：免費、零後端成本、與 Google Sheet 原生整合
- **為何選 GitHub Pages**：免費靜態部署、版本控制整合
- **為何不用資料庫**：店家不具備技術背景，Google Sheet 最易維護

### 已解決的關鍵問題

| 問題 | 原因 | 解法 |
|------|------|------|
| fetch 失敗（Failed to fetch）| Apps Script URL 舊版失效 | 重新部署取得新URL |
| 訂單找不到（找不到訂單錯誤）| Sheet 將訂單編號存為數字，比對字串失敗 | 訂單編號改為 `Z` 開頭（字母開頭不會被存為數字）|
| LOGO 不顯示 | JPG白底在紅色背景上整張變白 | Python去背轉透明PNG |
| Google Drive圖片無法顯示 | CORS跨來源請求被擋 | 改存於GitHub repo |
| main/develop分支不同步 | 一直在develop提交未合併main | 手動merge develop→main |

### 版本控制
- 工具：GitHub Desktop + GitHub Releases
- 分支策略：`develop`（開發）→ `main`（正式）
- Release 標記方式：GitHub Releases（因本機無 git 指令）
