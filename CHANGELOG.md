## v1.9 — 庫存管理系統（2026/06/24）

### v1.9 Step1 — 下單自動扣庫存

#### 新增
- `deductStock()` 正式啟用（原已定義但未呼叫）
- 套餐自動拆解成單品後扣庫存（依 `SET_DECOMPOSE_BACKEND`）
- 允許庫存扣成負數，讓店家看出實際超賣數量（-1 = 超賣1份）
- `submitOrder()` 在訂單寫入後立即呼叫 `deductStock(data.orderItems)`
- 庫存扣減在 `LockService` 保護範圍內執行，防止並發競態

#### 規則確認
- 套餐拆解扣單品（A套餐 = dongpo×1 + sun×1）
- 取消訂單不自動補回庫存
- 補貨由店家直接修改 Google Sheet G欄

- `Code.gs`

## v1.6.6 — 安全強化修正（2026/06/24）

### 修改（扣哥安全審查後修正）
- XSS 防護：`esc()` 函式強化，涵蓋雙引號與單引號

## v1.6.5 — 後台登入驗證（2026/06/24）

### 新增
- UUID Token 登入機制，取代原 `prompt` 明碼比對
- 密碼改存於 `Script Properties`（`ADMIN_PASSWORD`），不硬寫在程式碼
- `login()` / `verifyToken()` / `logout()` 後端函式
- Token 有效期 24 小時，過期自動要求重新登入
- 單一裝置限制：新登入會使舊 Token 失效（`CURRENT_TOKEN_` 機制）
- `getOrders` / `updateOrder` API 均需 Token 驗證
- 前台 Token 存於 `localStorage`，不出現於 URL 或 Console

## 開發過程重要決策記錄

### 技術選型
- **為何選 Google Apps Script**：免費、零後端成本、與 Google Sheet 原生整合
- **為何選 GitHub Pages**：免費靜態部署、版本控制整合
- **為何不用資料庫**：店家不具備技術背景，Google Sheet 最易維護

### 已解決的關鍵問題

| 問題 | 原因 | 解法 |
|------|------|------|
| fetch 失敗（Failed to fetch）| Apps Script URL 舊版失效 | 重新部署取得新URL |
| 訂單找不到（找不到訂單錯誤）| Sheet 將訂單編號存為數字，比對字串失敗 | 訂單編號改為 `Z` 開頭（字母開頭不會被存為數字）|
| LOGO 不顯示 | JPG白底在紅色背景上整張變白 | Python去背轉透明PNG |
| Google Drive圖片無法顯示 | CORS跨來源請求被擋 | 改存於GitHub repo |
| main/develop分支不同步 | 一直在develop提交未合併main | 手動merge develop→main |
| 今日統計返回零 | ISO時間格式與斜線格式不一致 | `formatDateField()` 統一輸出格式 |
| deductStock 未執行 | 前台從未傳送 orderItems | v1.7 改送結構化 orderItems，v1.9 正式啟用 |

### 版本控制
- 工具：GitHub 網頁介面（已停用 GitHub Desktop，避免多裝置同步問題）
- 分支策略：`develop`（開發）→ PR → `main`（正式），合併前須確認 Files Changed diff
- Release 標記方式：GitHub Releases
