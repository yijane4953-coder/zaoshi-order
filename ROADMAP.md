## 已完成

### v1.9 — 庫存管理系統（2026/06/24）

**Step1 — 下單自動扣庫存：**
- `deductStock()` 正式啟用
- 套餐自動拆解成單品扣庫存（A套餐 = 東坡肉×1 + 香滷嫩筍×1）
- 允許負數庫存，讓店家看出超賣數量
- 在 `LockService` 保護範圍內執行，防止並發競態
- 取消訂單不自動補回庫存

**Step2 — 後台低庫存／超賣警示：**
- 後台頂部「⚠️ 庫存警示」區塊（有問題才顯示）
- 三層分類：🔴 已超賣 / ⚠️ 已售完 / 🟠 低庫存（≤3份）
- 每次載入訂單後自動更新

**Step3 — 庫存管理頁面：**
- 後台新增「📦 目前庫存」區塊，預設收合
- 四個單品庫存卡片（emoji、名稱、數量、狀態標籤）
- 位置：庫存警示下方、統計卡上方

**已確認規則：**
- 庫存單位：套餐拆解扣單品
- 允許負數（-1 = 超賣1份）
- 低庫存門檻：≤ 3
- 補貨方式：直接修改 Google Sheet G10:G13

## 開發原則

1. **不重構**：每次只針對特定功能修改，不動已正常運作的部分
2. **MVP 優先**：先讓功能可用，再優化體驗
3. **手機優先**：所有 UI 以手機操作為主要考量
4. **低成本**：優先使用免費工具，確定長期需求再投入費用
5. **可維護**：程式碼保持可讀，重要設定集中在檔案頂部

## 版本命名規則

```
v主版本.次版本.修補版本

主版本：重大功能或架構改變
次版本：新增功能
修補版本：Bug 修正或小調整

範例：
v1.0   → 初次上線
v1.1   → 新增自取時段
v1.9   → 庫存管理系統
```
