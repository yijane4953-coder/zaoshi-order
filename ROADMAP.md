### v1.8 — 第四階段：庫存系統

**功能：**
- 從 Google Sheet 商品設定分頁讀取庫存
- 庫存 > 0：可加入購物車
- 庫存 = 0：顯示「售完」，無法點選
- 套餐：包含任一售完品項則套餐也顯示售完
- 送出訂單後自動扣除庫存（啟用既有 `deductStock()`，第三階段已完成資料管線、但刻意未啟用）
- 防止超賣（庫存不足時拒絕下單）

**實作重點：**
- 前台載入時需先 fetch 商品設定
- `deductStock()` 函式已存在，第三階段已完成「結構化商品清單」資料管線，本階段需補上「扣庫存前檢查庫存是否足夠」的判斷，與寫入訂單同一原子操作內完成
- **務必複用第三階段 `submitOrder()` 已建立的 `LockService` 鎖定機制**：扣庫存涉及「讀取現有庫存→判斷足夠→扣減寫回」，與訂單編號生成是同類型的競態條件問題（第三階段已實際測試重現並修正過，此處直接沿用相同模式，不需重新踩一次坑）

**修改檔案：** `index.html`、`Code.gs`

**開發建議：** 於 `develop` 分支進行，比照第三階段經驗：開發完成後用關鍵字搜尋確認沒有遺漏既有功能，並針對「並發扣庫存」情境實際測試（模擬兩筆訂單同時搶購最後一件庫存），確認不會超賣，測試通過後再 merge 回 `main`。

## 開發原則

1. **不重構**：每次只針對特定功能修改，不動已正常運作的部分
2. **MVP 優先**：先讓功能可用，再優化體驗
3. **手機優先**：所有 UI 以手機操作為主要考量
4. **低成本**：優先使用免費工具，確定長期需求再投入費用
5. **可維護**：程式碼保持可讀，重要設定集中在檔案頂部

```

主版本：重大功能或架構改變

修補版本：Bug 修正或小調整

v1.1   → 新增自取時段
v1.3.1 → 修補圖片功能（1.3的子版本）
```
